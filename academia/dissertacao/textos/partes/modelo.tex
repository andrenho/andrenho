\section{eProfile}

O eProfile é um gerenciador de perfis voltado para ambientes distribuídos, que infere dados de perfil através da análise das trilhas de uma entidade (disponibilizadas por um gerenciador de trilhas), seguindo regras definidas pelas próprias entidades, e disponibilizado os dados inferidos para serem utilizados por múltiplas aplicações.

A Figura~\ref{fig:modelo} apresenta o modelo, que é organizado em três partes.

\begin{figure}
  \centering\includegraphics[width=0.5\textwidth]{imagens/modelo.png}
  \caption{Modelo do eProfile}
  \label{fig:modelo}
\end{figure}

O eProfile é o modelo apresentado neste artigo. Ele interage com o gerenciador de trilhas através de uma interface padronizada, de onde busca as informações necessárias para a inferência dos perfis.

Para que o eProfile possa operar, é necessário que o mesmo esteja conectado a um gerenciador de trilhas que implemente uma interface padrão. O gerenciador de trilhas é o módulo responsável por armazenar as trilhas, fornecendo uma interface que permita a consulta de um conjunto destas trilhas dentro de parâmetros específicos (por exemplo, entidade ou data). O gerenciador de trilhas deve fornecer as trilhas no formato especificado pelo eProfile.

Os clientes referem-se a dispositivos que se comunicam com o gerenciador de trilhas e com o eProfile. Estes clientes fazem o registro de seus eventos dentro do gerenciador de trilhas. O eProfile e o gerenciador de trilhas trabalham em conjunto, independente dos clientes, de modo a manter as informações de perfil preprocessadas dentro do eProfile.

Cada cliente é responsável por fornecer o conjunto de regras para o processamento de suas próprias trilhas. Desta forma, cada cliente pode registrar os eventos de histórico no formato que considerar pertinente, desde que as regras respeitem o formato utilizado.
