O eProfile é um gerenciador de perfis voltado para ambientes distribuídos, que infere dados de perfil através de inferências nas trilhas de uma entidade. A trilha é disponibilizada por um gerenciador de trilhas que se conecta ao eProfile, onde é feita uma mediação entre o formato de dados da trilha e do eProfile.

A figura tradicional do usuário na personalização é estendida pelo conceito de \emph{entidade}. Uma entidade é um conceito, não representando apenas pessoas mas também objetos que possam criar uma trilha e ter um perfil, como um veículo ou um dispositivo móvel. Uma entidade pode ser (i) simples, mapeando os dados de uma única entidade física, ou (ii) composta, onde refere-se a um grupo de entidades. Um exemplo de uma entidade composta seria uma frota de caminhões, onde cada caminhão é uma entidade simples que tem seu perfil único, e onde a frota tem um perfil também.

A ontologia de perfil do eProfile pode ser estendida pelas aplicações, de modo que as aplicações possam armazenar dados de perfil de acordo com um modelo de usuário próprio. Assim, as aplicações podem disponibilizar suas próprias regras para a inferência das informações relacionadas à sua própria estrutura de dados. Os dados inferidos ficam então disponíveis para serem consultados pelas entidades.

Como o eProfile é um modelo que infere informações de forma autônoma a respeito de dados armazenados em uma ontologia, a tecnologia de agentes autônomos foi escolhida para o desenvolvimento.
