O eProfile é um gerenciador de perfis voltado para ambientes distribuídos, que infere dados de perfil através da inferência das trilhas de uma entidade. A trilha é disponibilizada por um gerenciador de trilhas que se conecta ao eProfile, onde é feita uma mediação entre o formato de dados da trilha e do eProfile.

A figura tradicional do usuário na personalização é extendida aqui pelo conceito de \emph{entidade}. Uma entidade é um conceito mais amplo que o de um usuário, não representando apenas pessoas mas também objetos que possam criar uma trilha e ter um perfil, como um veículo ou um dispositivo móvel. Uma entidade pode também ser composta por um grupo de entidades como, por exemplo, uma frota de caminhões onde cada caminhão tem seu perfil único, e onde a frota tem um perfil também.

A ontologia de perfil do eProfile pode ser extendida pelas aplicações, de modo que as aplicações possam armazenar dados de perfil que interessem apenas a si mesmos. Assim, as aplicações podem disponibilizar suas próprias regras para a inferência das informações relacionadas à sua própria estrutura de dados. Os dados inferidos ficam então disponíveis para serem consultados pelas entidades.

Como o eProfile é um modelo onde é necessário inferir informações de forma autônoma a respeito de dados armazenados em uma ontologia, a tecnologia de agentes autônomos foi escolhida para o desenvolvimento.
