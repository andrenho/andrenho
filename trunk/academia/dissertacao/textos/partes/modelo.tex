\section{eProfile}

O eProfile é um gerenciador de perfis voltado para ambientes distribuídos, que infere dados de perfil através da análise das trilhas de uma entidade, seguindo regras definidas pelas próprias entidades, e disponibilizado os dados inferidos para serem utilizados por múltiplas aplicações.

A Figura~\ref{fig:modelo} apresenta o modelo, que é organizado em três partes.

\begin{figure}
  \includegraphics[width=1\textwidth]{imagens/modelo.png}
  \caption{Modelo do eProfile}
  \label{fig:modelo}
\end{figure}

O eProfile é o modelo apresentado neste artigo. Ele interage com o gerenciador de trilhas através de uma interface padronizada, de onde busca as informações necessárias para a inferência dos perfis.

Para que o eProfile possa operar, é necessário que o mesmo esteja conectado a um gerenciador de trilhas que implemente uma interface padrão. O gerenciador de trilhas é o módulo responsável por armazenar as trilhas, fornecendo uma interface que permita a consulta de um conjunto destas trilhas dentro de parâmetros específicos (por exemplo, entidade ou data). O gerenciador de trilhas pode usar qualquer ontologia para o armazenamento das trilhas, desde que forneça estas trilhas no formato especificado pelo eProfile.

Os clientes referem-se a dispositivos que se comunicam com o gerenciador de trilhas e com o eProfile. Estes clientes fazem o registro de seus eventos dentro do gerenciador de trilhas. O eProfile e o gerenciador de trilhas trabalham em conjunto, independente dos clientes, de modo a manter as informações de perfil pré-processadas dentro do eProfile.


\subsection{Modelagem}

O eProfile é um sistema orientado a agentes, onde a metodologia utilizada para a modelagem é a Prometheus\cite{prometheus}. Esta metodologia define um processo detalhado para a especificação, projeto, implementação e teste/depuração de sistemas de software orientados a agentes.\cite{prometheus-book}

\begin{figure}
  \includegraphics[width=1\textwidth]{imagens/prometheus.png}
  \caption{Processo de projeto segundo a metodologia Prometheus \cite{prometheus-book}}
  \label{fig:metodologia}
\end{figure}

A metodologia Prometheus, ilustrada na Figura~\ref{fig:metodologia} define que o primeiro passo da modelagem de um sistema é a identificação dos objetivos inciais do sistema. De acordo com o modelo, os objetivos são:

\begin{itemize}
  \item Gerenciar múltiplos perfis
  \item Buscar trilhas das entidades através da conexão com um gerenciador de trilhas
  \item Inferir dados de perfil através da análise das trilhas
  \item Aceitar requisições de informações de perfil pelos clientes
  \item Aceitar alterações nas regras a partir dos clientes
\end{itemize}
