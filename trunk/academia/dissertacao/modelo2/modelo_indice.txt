2. Modelo

O eProfile é um gerenciador de perfis orientado a agentes para ambientes distribuídos. Ele é projetado para trabalhar em conjunto com um gerenciador de trilhas genérico, da qual busca dados que são utilizados para inferir informações relevantes para a formação de perfil. As regras utilizadas para inferência de perfil podem ser tanto pré-determinadas quanto especificadas pelos clientes.

O eProfile é o modelo apresentado neste artigo. Ele interage com o gerenciador de trilhas através de uma interface padronizada, de onde busca as informações necessárias para a inferência dos perfis.

Para que o eProfile possa operar, é necessário que o mesmo esteja conectado a um gerenciador de trilhas que implemente uma interface padrão. O gerenciador de trilhas é o módulo responsável por armazenar as trilhas, fornecendo uma interface que permita a consulta de um conjunto destas trilhas dentro de parâmetros específicos (por exemplo, entidade ou data). O gerenciador de trilhas pode usar qualquer ontologia para o armazenamento das trilhas, desde que forneça estas trilhas no formato especificado pelo eProfile.

Os clientes referem-se a dispositivos que se comunicam com o gerenciador de trilhas e com o eProfile. Estes clientes fazem o registro de seus eventos dentro do gerenciador de trilhas. O eProfile e o gerenciador de trilhas trabalham em conjunto, independente dos clientes, de modo a manter as informações de perfil pré-processadas dentro do eProfile.


2.1 Modelo de Dados

O perfil de uma entidade diz respeito a um conjunto de dados que contém todas as informações de perfil daquela entidade. O perfil é composto de duas partes:

 * dados pré-definidos: referem-se a uma ontologia de dados pré-definida pelo eProfile. A ontologia ainda está pendente de definição. Estas informações serão utilizadas pelos algoritmos de inferência pré-definidos no eProfile;

 * dados livres: é um conjunto de dados em formato livre, que podem tanto se referir a dados de simples consulta ou informações utilizadas para inferência realizada por algoritmos customizados.

Os campos podem ser de dois tipos:

 * informados: são campos de cadastro simples, onde a informação gravada será consultada posteriormente;

 * inferidos: são campos que são calculados a partir de informações buscadas do gerenciados de trilhas. Estes campos estarão vinculados a um conjunto de regras, que definirá como e quando as informações serão atualizadas.

2.2 Inferência

O eProfile é um gerenciador de perfis que oferece, além da opção do armazenamento de dados estáticos, a possibilidade de inferir dados de perfis a partir de informações registradas no gerenciador de trilhas. A inferência pode ser de dois tipos:

 * pré-definida: refere-se a um conjunto de algoritmos pré-definidos que fazem parte do eProfile. A saída destes algoritmos será gravada apenas nos dados pré-definidos do perfil. O conjunto de algoritmos a serem disponibilizados ainda está em definição;

 * customizada: refere-se a um conjunto de regras enviado pelos clientes, onde os próprios clientes definem as regras de inferência e a frequência de atualização. Os clientes podem utilizar o conjunto de dados livres, uma vez que utilizando o próprio algoritmo, são livres para criar novos campos. 

A forma como as regras customizadas serão enviadas ao servidor está ainda em aberto, onde foram discutidas três possibilidades: upload de código para rodar no servidor, envio de um algoritmo através de uma linguagem a ser criada, e envio de regras de um sistema especialista (ou um conjunto destas possibilidades).

A freqüência de execução dos algoritmos de inferência fazem parte da definição das próprias regras. Como o eProfile irá consultar os dados no gerenciador de trilhas, a inferência pode ser executada independente dos clientes estarem conectados ou não. Desta forma, quando houver a necessidade do cliente buscar o dado inferido, a inferência já terá sido executada previamente e o dado está prontamente disponível.



2.3 Comunicação com os clientes

A inferência das informações de perfil ocorrerá à parte do cliente. O cliente fica responsável apenas por informar o quão freqüente ele deseja que as informações sejam atualizadas, e o eProfile irá realizar a inferência mesmo que o cliente esteja desconectado.

O cliente pode solicitar as informações de perfil de duas formas:

 1. através de uma chamada, onde ele solicita as informações;

 2. através de uma inscrição, onde o cliente se inscreve no servidor e
informa que deseja ser informado cada vez que um determinado campo
mudar. Assim, quando o campo é alterado, o servidor contata o cliente
e o informa da mudança.
